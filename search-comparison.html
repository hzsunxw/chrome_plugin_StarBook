🔍 智能问答搜索方法对比

🔤 关键词匹配搜索

当前实现的方法

工作原理：

  1. 提取用户问题中的关键词
  2. 在书签的title、summary、tags等字段中查找匹配
  3. 根据匹配字段的权重计算相关度分数
  4. 按分数排序返回结果

示例：

问题："pdf怎么转成ppt"

关键词：["pdf", "转成", "ppt", "PowerPoint"]

匹配逻辑：检查每个书签是否包含这些关键词

✅ 优点：

  • 响应速度快（毫秒级）
  • 不消耗API费用
  • 离线可用
  • 结果可预测
  • 实现简单

❌ 缺点：

  • 无法理解语义
  • 同义词支持有限
  • 无法处理复杂查询
  • 依赖精确匹配
  • 无法推理

🤖 AI智能搜索

新增的AI增强方法

工作原理：

  1. 将用户问题和所有书签数据发送给AI
  2. AI分析问题意图和书签内容的语义
  3. AI评估每个书签的相关度并给出理由
  4. 返回智能排序的结果和推荐理由

示例：

问题："pdf怎么转成ppt"

AI分析：理解用户需要文档格式转换工具

智能匹配：找到相关工具，即使描述中没有直接包含关键词

✅ 优点：

  • 理解语义和意图
  • 处理复杂查询
  • 提供推荐理由
  • 支持自然语言
  • 智能推理能力

❌ 缺点：

  • 响应较慢（秒级）
  • 消耗API费用
  • 需要网络连接
  • 结果可能不稳定
  • 实现复杂

🎯 使用场景建议

适合关键词搜索的场景：

  • 🔍 精确查找：知道具体的技术名词或品牌名
  • ⚡ 快速查询：需要即时响应的场景
  • 💰 成本敏感：不想消耗API额度
  • 📱 离线使用：网络不稳定的环境
  • 🎯 简单需求：直接的关键词匹配就能满足

示例查询：

  • "JavaScript教程"
  • "PDF工具"
  • "React文档"

适合AI搜索的场景：

  • 🧠 语义查询：用自然语言描述需求
  • 🔄 复杂需求：需要理解上下文和意图
  • 💡 发现相关：找到意想不到的相关内容
  • 📚 学习探索：不确定具体要找什么
  • 🎨 创意查询：需要AI的推理和联想

示例查询：

  • "我想学习前端开发，从哪里开始？"
  • "如何提高工作效率？"
  • "有什么好的设计灵感网站？"

⚙️ 混合策略

当前实现的智能策略：

  1. 优先AI搜索：如果配置了AI且开启智能搜索，优先使用AI
  2. 自动降级：AI搜索失败时自动降级到关键词搜索
  3. 用户选择：用户可以在AI配置中关闭智能搜索
  4. 最佳体验：结合两种方法的优势，提供最佳搜索体验
// 搜索策略伪代码 if (hasAIConfig && enableSmartSearch) { try { return await aiSmartSearch(question); } catch (error) { return await keywordSearch(question); // 降级 } } else { return await keywordSearch(question); }
